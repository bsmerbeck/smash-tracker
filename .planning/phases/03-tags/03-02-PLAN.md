<objective>
Deliver the first user-facing vertical slice of Phase 3: a user can tag a whole match with preset and freeform custom tags directly on the VOD Manager's metadata card, and remove tags with one click. This satisfies TAG-01 (match tagging) and TAG-03 (match preset list).

Purpose: Turn the deployed tag schema (03-01) into a real capability the user can see and use. Builds the two shared tag primitives — the `tags.ts` logic module and the `TagAddCombobox` component — that 03-03 (note tagging) and 03-04 (filtering) reuse, so they are built once here.

Output: `apps/web/src/lib/tags.ts` (preset lists + pure helpers, unit-tested), a reusable `TagAddCombobox` (preset → custom → create), match-tag chips wired into `SelectedMatchMeta`'s view state with immediate add/remove, and the carry-through plumbing (`buildUpdateInput` + `SelectedMatchMeta.onSubmit`) that keeps tags and every other field mutually intact. All new copy ships across the 6 locales.

MVP-mode note: Tasks 1-2 build the shared logic + combobox; Task 3 completes the slice — after Task 3 a real user can tag and untag a match, which they could not do before. The combobox and lib are genuine shared primitives (reused verbatim by 03-03/03-04), not horizontal scaffolding.
</objective>

## Phase Goal

**As a** SSBU player with a growing library of recorded VODs, **I want to** tag matches and individual timestamp notes with preset and freeform tags and filter my VOD list by those tags, **so that** my footage stays organized the way I actually think about my play and I can quickly narrow to what I'm looking for.
